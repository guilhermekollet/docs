## Descrição

Este endpoint retorna incidentes registrados relacionados aos monitores (agentes), como desconexões, falhas de sincronização, erros de autenticação, entre outros.

## Autenticação

<ParamField header="Authorization" type="string" required>
  Bearer token do usuário autenticado.
  
  **Formato:** `Bearer {access_token}`
</ParamField>

## Parâmetros de Query

<ParamField query="page" type="number" default="1">
  Número da página para paginação.
</ParamField>

<ParamField query="limit" type="number" default="10">
  Quantidade de incidentes por página.
</ParamField>

<ParamField query="monitor_id" type="string">
  Filtrar por ID específico do monitor.
</ParamField>

<ParamField query="status" type="string">
  Filtrar por status do incidente.
  
  **Valores aceitos:** `open`, `resolved`, `investigating`
</ParamField>

<ParamField query="incident_type" type="string">
  Filtrar por tipo de incidente.
  
  **Valores aceitos:** `disconnection`, `sync_error`, `auth_failed`, `timeout`
</ParamField>

## Resposta de Sucesso

<ResponseField name="data" type="array">
  Array contendo os incidentes de monitores.
  
  <Expandable title="Objeto Incidente">
    <ResponseField name="id" type="string">
      ID único do incidente (UUID).
    </ResponseField>
    
    <ResponseField name="monitor_id" type="string">
      ID do monitor afetado.
    </ResponseField>
    
    <ResponseField name="monitor_name" type="string">
      Nome do monitor.
    </ResponseField>
    
    <ResponseField name="incident_type" type="string">
      Tipo do incidente.
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Status atual.
    </ResponseField>
    
    <ResponseField name="severity" type="string">
      Nível de severidade.
    </ResponseField>
    
    <ResponseField name="error_message" type="string">
      Mensagem de erro detalhada.
    </ResponseField>
    
    <ResponseField name="affected_groups" type="number">
      Quantidade de grupos afetados.
    </ResponseField>
    
    <ResponseField name="detected_at" type="string">
      Data de detecção.
    </ResponseField>
    
    <ResponseField name="resolved_at" type="string">
      Data de resolução (null se não resolvido).
    </ResponseField>
  </Expandable>
</ResponseField>

## Exemplos de Código

<RequestExample>

```bash cURL
curl --request GET \
  --url 'https://api.monitora.com/incidents/monitors?status=open' \
  --header 'Authorization: Bearer SEU_ACCESS_TOKEN'
```

```javascript JavaScript
const params = new URLSearchParams({
  status: 'open',
  page: '1',
  limit: '10'
});

const response = await fetch(
  `https://api.monitora.com/incidents/monitors?${params}`,
  {
    headers: {
      'Authorization': 'Bearer SEU_ACCESS_TOKEN'
    }
  }
);

const data = await response.json();
```

```python Python
import requests

url = "https://api.monitora.com/incidents/monitors"
headers = {
    "Authorization": "Bearer SEU_ACCESS_TOKEN"
}
params = {
    "status": "open",
    "page": 1,
    "limit": 10
}

response = requests.get(url, headers=headers, params=params)
print(response.json())
```

## Tipos de Incidente

<AccordionGroup>
  <Accordion title="disconnection" icon="plug-circle-xmark">
    Monitor desconectado do WhatsApp - requer reconexão.
  </Accordion>

  <Accordion title="sync_error" icon="arrow-rotate-right">
    Erro na sincronização de dados dos grupos.
  </Accordion>

  <Accordion title="auth_failed" icon="key">
    Falha de autenticação - token expirado ou inválido.
  </Accordion>

  <Accordion title="timeout" icon="clock">
    Timeout nas requisições - monitor não responde.
  </Accordion>
</AccordionGroup>

## Notas Importantes

<Warning>
  Incidentes de monitores podem afetar múltiplos grupos. Resolva-os rapidamente para evitar perda de dados.
</Warning>
