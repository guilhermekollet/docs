## Descrição

Este endpoint retorna uma lista de incidentes registrados relacionados a links do WhatsApp (grupos e canais), permitindo monitorar problemas como links quebrados, expirados ou removidos.

## Autenticação

<ParamField header="Authorization" type="string" required>
  Bearer token do usuário autenticado.
  
  **Formato:** `Bearer {access_token}`
</ParamField>

## Parâmetros de Query

<ParamField query="page" type="number" default="1">
  Número da página para paginação.
</ParamField>

<ParamField query="limit" type="number" default="10">
  Quantidade de incidentes por página.
</ParamField>

<ParamField query="status" type="string">
  Filtrar por status do incidente.
  
  **Valores aceitos:** `open`, `resolved`, `investigating`
</ParamField>

<ParamField query="severity" type="string">
  Filtrar por severidade.
  
  **Valores aceitos:** `low`, `medium`, `high`, `critical`
</ParamField>

<ParamField query="start_date" type="string">
  Data inicial para filtro (ISO 8601).
</ParamField>

<ParamField query="end_date" type="string">
  Data final para filtro (ISO 8601).
</ParamField>

## Resposta de Sucesso

<ResponseField name="data" type="array">
  Array contendo os incidentes encontrados.
  
  <Expandable title="Objeto Incidente">
    <ResponseField name="id" type="string">
      ID único do incidente (UUID).
    </ResponseField>
    
    <ResponseField name="link_id" type="string">
      ID do link afetado.
    </ResponseField>
    
    <ResponseField name="url" type="string">
      URL do link do WhatsApp.
    </ResponseField>
    
    <ResponseField name="incident_type" type="string">
      Tipo do incidente (broken, expired, removed, invalid).
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Status atual (open, resolved, investigating).
    </ResponseField>
    
    <ResponseField name="severity" type="string">
      Nível de severidade (low, medium, high, critical).
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Descrição detalhada do incidente.
    </ResponseField>
    
    <ResponseField name="detected_at" type="string">
      Data de detecção (ISO 8601).
    </ResponseField>
    
    <ResponseField name="resolved_at" type="string">
      Data de resolução (ISO 8601, null se não resolvido).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  Metadados da paginação.
</ResponseField>

## Tipos de Incidente

<AccordionGroup>
  <Accordion title="broken" icon="link-slash">
    Link quebrado - retorna erro ao acessar (404, 500, etc).
  </Accordion>

  <Accordion title="expired" icon="clock">
    Link expirado - grupo foi desativado ou link temporário expirou.
  </Accordion>

  <Accordion title="removed" icon="trash">
    Grupo removido - o grupo não existe mais no WhatsApp.
  </Accordion>

  <Accordion title="invalid" icon="circle-xmark">
    Link inválido - formato incorreto ou URL malformada.
  </Accordion>
</AccordionGroup>

## Níveis de Severidade

| Severidade | Descrição | Ação Recomendada |
|------------|-----------|------------------|
| **Low** | Incidente de baixo impacto | Monitorar |
| **Medium** | Impacto moderado | Investigar em 24h |
| **High** | Alto impacto | Corrigir em 4h |
| **Critical** | Impacto crítico | Ação imediata |
