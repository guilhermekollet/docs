## Convenções da API

### Formatos de Dados

- **Datas**: ISO 8601 format (`2026-01-09T10:30:00Z`)
- **IDs**: Strings com prefixo do tipo (`acc_123`, `cmp_456`)
- **Valores monetários**: Inteiros em centavos
- **Booleanos**: `true` ou `false`

### Paginação

Endpoints que retornam listas suportam paginação:

```json
{
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 100,
    "total": 523,
    "has_more": true
  }
}
```

**Parâmetros:**
- `page`: Número da página (padrão: 1)
- `limit`: Itens por página (padrão: 100, máx: 1000)

### Ordenação

Use o parâmetro `sort`:

```
GET /v1/accounts?sort=-created_at
```

- Prefixo `-` para ordem decrescente
- Sem prefixo para ordem crescente

### Filtros

Filtre resultados com parâmetros de query:

```
GET /v1/accounts?status=active&created_after=2026-01-01
```

### Expansão de Recursos

Use `expand` para incluir recursos relacionados:

```
GET /v1/accounts/acc_123?expand=subscription,credits
```

### Idempotência

Use o header `Idempotency-Key` para operações críticas:

```http
POST /v1/payments
Idempotency-Key: unique-key-123
```

## Exemplos Rápidos

### Listar Contas

```bash
curl --request GET \
  --url 'https://api.example.com/v1/accounts?limit=10' \
  --header 'Authorization: Bearer YOUR_API_KEY'
```

### Criar Conta

```bash
curl --request POST \
  --url https://api.example.com/v1/accounts \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "John Doe",
    "email": "john@example.com",
    "type": "business"
  }'
```

### Atualizar Conta

```bash
curl --request PATCH \
  --url https://api.example.com/v1/accounts/acc_123 \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "John Smith"
  }'
```

### Deletar Conta

```bash
curl --request DELETE \
  --url https://api.example.com/v1/accounts/acc_123 \
  --header 'Authorization: Bearer YOUR_API_KEY'
```
